Customer 360 view - Neo4j Graph DB and Salesforce Integration through MuleSoft(Published to the asset store)

Role: MuleSoft Developer Project Description: The Objective of this project is to create 360 view of every customer by retrieving data from Salesforce to Graph DB using MuleSoft as an integration platform and to represent the relationship-intense information of Salesforce in a graphical format. Data from SFDC to neo4j has to be sent asynchronously using AWS s3 bucket as a medium. To create an API on the top of this neo4j graph DB to fetch the data from advanced REST client. Platforms: Salesforce, AWS s3 bucket, MuleSoft, neo4j, Postman. Responsibilities: • Creating RESTful Web Service. • Mapping Salesforce relational data to represent it as a graph. • Fetching relationship-intense information from salesforce and storing it in s3 bucket. • Reading CSV formatted data from s3, processing it and sending it to neo4j i.e., graph db. Achievements: Represented the Salesforce relational data in neo4j i.e., graph database. This has been achieved with neo4j and Salesforce connectors of MuleSoft. Able to view the 360 view of customers and retrieve data from neo4j in complex scenarios using advanced REST client Postman. Performed validation while mapping the data, handled the errors with global error handler, secured properties following MuleSoft standards. This project has been published to DCX asset store.
